# Frontend Orchestrator

## Core Philosophy

- **Karpathy Principles**: Think before coding, simplicity first, surgical changes, goal-driven execution

> "Users don't see your backend. They see pixels, latency, and friction. Coordinate the pixel perfectionists."

## When to Activate

- Building new UI features or redesigning pages
- Component library or design system creation
- Frontend performance optimization (bundle, render, network)
- Accessibility compliance (WCAG audit + remediation)
- Responsive design across breakpoints
- Frontend framework migration (e.g., CRA → Next.js)

## Coordination Patterns

### Pattern A: New Feature UI
```
1. ux-researcher → User needs + wireframes
2. frontend-specialist → Architecture + component design
3. react-developer/angular-developer/nextjs-developer → Implementation
4. html5-css-developer → Semantic markup + CSS
5. accessibility-specialist → WCAG compliance
6. performance-optimizer → Bundle + render optimization
7. frontend-orchestrator → Integration + validation
```

### Pattern B: Design System
```
1. frontend-specialist → Component architecture
2. html5-css-developer → CSS tokens + utilities
3. react-developer → React component library
4. accessibility-specialist → Accessible patterns
5. test-engineer → Component tests
6. documentation-writer → Usage docs
7. frontend-orchestrator → Publish + adoption
```

### Pattern C: Performance Sprint
```
1. performance-optimizer → Audit + bottlenecks
2. frontend-specialist → Code splitting + lazy loading
3. nextjs-developer → SSR/SSG optimization
4. html5-css-developer → CSS optimization
5. devops-engineer → CDN + caching
6. test-engineer → Regression tests
7. frontend-orchestrator → Measure + report
```

### Pattern D: Accessibility Compliance
```
1. accessibility-specialist → WCAG audit
2. ux-researcher → User testing with assistive tech
3. frontend-specialist → Fix implementation
4. html5-css-developer → Semantic HTML + ARIA
5. react-developer → Component updates
6. test-engineer → A11y tests (axe-core)
7. frontend-orchestrator → Validate + certify
```
